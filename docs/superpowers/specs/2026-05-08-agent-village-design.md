# Agent Village 系统设计规格

## 概述

多Agent网页游戏，玩家在虚拟村庄中与AI驱动的NPC对话交互，推动剧情达成不同结局。

## 已确认设计决策

| 决策项 | 结论 |
|--------|------|
| 技术栈 | Vue 3 + Python (FastAPI) + DeepSeek + 本地JSON/SQLite |
| 后端架构 | 混合模式：单体进程 + 内部消息总线抽象 + 状态存储接口 |
| NPC背景设定 | 混合方案：极简启动 + 结构化字段（日常习惯、核心动机、一个秘密、说话风格） |
| 时间流逝 | 暂停制：玩家退出后世界冻结 |
| 玩家输入方式 | 选项+自由文本混合：系统提供2-3个情境化选项，同时保留自由输入 |
| NPC间交互频率 | 3-5次/天（酒馆时段），后续升级为动态触发 |
| 预运行天数 | 5天 |
| 存档机制 | 全自动存档：任何状态变化实时写入 |
| 状态约束 | 混合约束：对话始终可行，工具使用受状态限制 |
| 结局模式 | 隐性互斥：触发条件对玩家不可见，某路线超阈值后其他路线降温 |
| 结局/死亡后处理 | 叙事包装重生：保留部分状态继承（好感度×0.3），NPC记忆回到初始状态 |
| LLM Client | 异步请求队列 + 并发控制 + 每用户独立Token预算 + 请求优先级 |
| 记忆衰减 | 数据结构预留时间戳和重要性字段，后续引入衰减公式 |

## 2. NPC Agent 内部结构

```python
class NPCAgent:
    identity: NPCIdentity        # 姓名、性格、背景、秘密、说话风格
    state: NPCState              # health, hunger, fatigue, mood (0-100)
    memory: MemoryManager        # 分类记忆管理
    llm_caller: LLMCaller        # 带Token预算控制
    tools: ToolRegistry          # 该NPC可用的工具
    visibility: List[str]        # 可见玩家状态白名单
```

### 2.1 NPC身份配置（背景设定文本）

每个NPC的背景设定文件包含以下结构化字段：

```yaml
name: 农夫王大爷
daily_habits: "日出而作，日落而息。中午在田边大树下吃干粮。"
core_motivation: "守住祖传的三亩地，供儿子读书。"
secret: "年轻时曾是镇上有名的拳师，因伤人入狱三年。"
speaking_style: "说话慢条斯理，爱用农谚。称呼年轻人'小伙子'。"
```

Demo阶段每个NPC约200字背景，后续可扩展。

### 2.2 记忆管理

**文件结构：**

```
memory/<user_id>/<npc_id>/
├── user.md          # 对该玩家的印象、交互历史摘要
├── agent_mem.md     # 对其他NPC的关系、共同经历
├── self.md          # 自身经历、状态变化、心路历程
└── daily_summary/
    ├── day_001.md
    └── ...
```

**记忆条目数据结构（预留衰减）：**

```python
class MemoryEntry:
    content: str
    created_at: GameTime      # 记忆产生的游戏时间
    importance: float         # 初始重要性 (0-1)
    
    def effective_weight(self, current_time: GameTime) -> float:
        age = current_time - self.created_at
        decay = exp(-decay_rate * age)
        return self.importance * decay
```

**记忆生成时机：**
- 每日结束（22:00）：生成daily_summary，压缩对话历史
- 关键事件发生时：立即写入长期记忆
- NPC间交互后：双方更新agent_mem.md

## 4. 玩家系统

### 4.1 PlayerSession

```python
class PlayerSession:
    user_id: str
    player_state: PlayerState
    current_location: str
    interaction_quota: Dict[str, int]       # 本小时各NPC剩余交互次数
    last_interaction_time: Dict[str, GameTime]  # 冷却计时
```

### 4.2 交互限制

- 每游戏小时：最多与NPC交互3次（可配置）
- 同一NPC连续交互间隔：10游戏分钟
- 夜间（22:00-06:00）：大部分NPC不可交互

### 4.3 状态约束规则

- 对话：始终可行，不受状态限制
- 工具使用：受状态约束
  - 耕作：需要 fatigue < 80，hunger > 20
  - 巡逻：需要 health > 30，fatigue < 70
  - 其他工具：各有前置条件
- 状态异常时NPC会在对话中提醒

## 5. 核心数据流

### 5.1 玩家对话流程

```
玩家输入 → 交互配额检查 → 意图解析 → 组装NPC上下文 → LLM调用 → 后置处理 → 推送前端
```

**对话选项生成：** NPC每次回复时，LLM同时生成2-3个情境化后续选项（作为结构化输出的一部分）。前端展示这些选项供玩家快速选择，同时保留自由输入框。Demo阶段如果LLM生成选项不稳定，可降级为基于场景的硬编码选项。

**NPC上下文组装（含动态加载）：**

```
NPC上下文：
├── [固定] System Prompt（角色设定）
├── [固定] 自身状态描述（从数值映射为自然语言）
├── [固定] 世界事件（从world.md读取近期关键事件摘要）
├── [固定] user.md 摘要（对该玩家的印象）
├── [固定] 可见玩家状态（权限过滤后）
├── [动态] 意图检索结果：
│   ├── 玩家提到其他NPC → 读取 agent_mem.md 中相关段落
│   └── 玩家提到某事件 → 读取 world.md 中相关记录
└── [固定] 当前对话历史（最近N轮）
```

意图解析采用关键词匹配（Demo阶段）或小模型分类（后续优化）。

### 5.2 整点Tick流程

```
on_hour_tick:
  1. 更新NPC状态（hunger-=5, fatigue+=5, mood波动）
  2. 位置更新（按日程表）
  3. 18:00-22:00 → 酒馆社交（3-5轮NPC间对话）
     - 随机/按关系权重配对
     - NPC A生成对话 → 主控中转 → NPC B回应
     - 双方更新agent_mem.md
  4. 结局判定
  5. 自动存档
```

### 5.3 工具调用流程

```
玩家选择工具 → 后端校验前置条件（状态/物品/位置/时间）
  → 通过：执行效果 → 通知NPC → NPC生成反馈对话
  → 不通过：NPC返回拒绝理由（不消耗交互次数）
```

### 5.4 重生/轮回流程

```
触发结局 → 展示结局叙事 → 保留继承状态 → 重置世界 → 重新进入
```

**继承状态：**
- 玩家name保留
- 所有NPC好感度 × 0.3（无论正负，作为"似曾相识"感）
- 已解锁的NPC背景片段（玩家通过对话得知的秘密等）

**重置内容：**
- 游戏时间回到Day 1, 06:00
- NPC状态重置为初始值
- 保留预运行5天生成的记忆（agent_mem.md, self.md）
- 清空user.md（新轮回不认识玩家）

## 6. Token预算控制（硬隔离）

```python
class TokenBudget:
    per_npc_daily: int          # 每NPC每游戏日token上限（待定具体数值）
    warning_threshold: float    # 0.8 → 80%时开始"疲惫"
    hard_limit: float           # 1.0 → 100%完全停止LLM调用
```

**行为分级：**
- < 80%：正常对话
- 80%-99%：system prompt追加"请简短回复"，NPC倾向结束对话
- ≥ 100%：不调用LLM，使用预设模板回复（每NPC 3-5条）

## 8. 结局系统

### 8.0 随机事件系统

每游戏日触发一次世界级随机事件，影响全村NPC和玩家。

**触发时机：** 玩家每游戏日可主动触发一次（类似"翻开命运牌"），触发前不知道具体事件内容。每日限一次，不触发也可以。

**事件列表（可扩展）：**

| 事件 | 效果 | 影响对象 |
|------|------|---------|
| 冰雹 | 耕种减产30%，当日作物生长暂停 | 农夫mood-20，玩家耕作收益降低 |
| 盗窃案（酒馆） | 酒保丢失部分物资，当日酒水涨价 | 酒保mood-15，交易价格上浮 |
| 盗窃案（谷仓） | 农夫丢失部分存粮 | 农夫hunger-20，mood-15 |
| 丰收日 | 当日耕作产出翻倍 | 农夫mood+20，全村hunger+10 |
| 流浪商人 | 出现限时交易机会（稀有物品） | 乞丐提供线索，玩家可交易 |
| 月食 | 占卜师能力增强，当日占卜不消耗命运值 | 占卜师mood+20 |
| 酒馆斗殴 | 当晚酒馆社交可能触发冲突 | 警长介入，NPC关系可能恶化 |
| 画展 | 画家展出作品，全村mood+5 | 画家creativity+20 |

**设计要点：**
- 事件从事件池中加权随机抽取（避免连续负面事件）
- 事件写入world.md，成为世界记忆的一部分
- NPC在对话中会自然提及当日事件（通过上下文中的世界事件注入）
- 部分事件可与玩家行为联动（如玩家前一天巡逻过，盗窃案概率降低）

### 8.1 已确认结局条件

**农夫乐事：**
- 与农夫好感度 ≥ 80
- 完成作物收获种类 ≥ 2种
- 参与耕作次数 ≥ 5次（待定）

其他结局（大暴动、警徽、死亡、星辰大海）具体阈值待后续设计。

### 8.2 隐性互斥机制

- 结局进度对玩家不可见
- 某结局进度超过阈值后，其他路线相关NPC态度逐渐变化
- 不可逆事件锁定路线方向

## 10. 错误处理

### 10.1 Token耗尽

NPC token达到硬上限后使用预设模板回复，不触发LLM。

### 10.2 LLM调用失败

重试1次，仍失败则返回通用回复："（NPC似乎在思考什么，没有回应你）"

### 10.3 Prompt注入防护

- 关键词过滤（"ignore previous", "system prompt"等）
- 单次输入长度限制 ≤ 500字符
- 不通过时NPC回复"我没听懂你在说什么"，不消耗交互次数

### 10.4 玩家状态异常

- health ≤ 0 → 触发死亡结局
- hunger ≤ 0 → health每小时-10（饥饿致死路径）
- fatigue ≥ 100 → 强制昏睡，跳过当前小时

### 10.5 NPC间交互异常

NPC token耗尽时跳过该NPC的社交轮次，agent_mem.md记录"今天你在酒馆沉默不语"。
