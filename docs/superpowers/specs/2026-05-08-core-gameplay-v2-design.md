# Agent Village 核心玩法 v2 — 洋葱式秘密网

## 概述

玩家置身于一个由AI驱动的虚拟村庄。核心乐趣不是"通关"，而是**观察NPC社会的自发演化**和**作为隐形推手注入事件**，看世界如何回应。玩家每天有1次发布事件的权利，可以随机抽取或自定义；NPC之间自主交互产生意外剧情；玩家通过对话和偷听引导剧情方向。没有固定结局，玩到不想玩为止。

## 1. 核心玩法循环

```
每天开始 → 玩家获得 1次"发布事件"权利
    ↓
├─ 随机事件卡：系统从事先配置的事件池中抽取，必定成功
├─ 自定义事件：玩家自由输入 → 系统评估颠覆性 → 摇骰子判定
│   ├─ 通过 → 事件按玩家意图发生
│   └─ 失败 → 事件以"搞耍"形式发生（事件发生但效果荒诞化）
│
├─ NPC之间自主交互（时段驱动 + 状态驱动 + 事件驱动）
│
└─ 玩家观察结果 + 与NPC对话互动 → 笔记簿自动记录摘要
    ↓
明天继续，无限循环
```

### 1.1 三股力量交织

| 力量 | 来源 | 说明 |
|------|------|------|
| 世界自发演化 | NPC间日常交互 | 玩家不干预也会发生，节奏较慢但持续 |
| 玩家注入事件 | 每日1次 | 玩家的"搅局"能力，催化剧情 |
| 玩家直接互动 | 对话引导 | 偷听、透露信息、引诱NPC向某方向变化 |

## 2. 事件注入系统

### 2.1 每日事件特权

每天1次，二选一：**随机事件卡** 或 **自定义事件**。

### 2.2 随机事件卡

从预先配置的事件池中加权随机抽取（避免连续负面）。抽到的卡必定成功，但玩家不知道会抽到什么。

事件池示例（可扩展）：
- 冰雹、盗窃案、丰收日、流浪商人、月食、酒馆斗殴、画展、谣言传播...

事件写入 world.md，成为世界记忆。NPC在后续对话中会自然提及。

### 2.3 自定义事件 + 骰子机制

**流程：**

```
玩家输入事件描述
    ↓
系统评估"颠覆性"（1-10分）
    ↓
颠覆性映射为骰子难度
    ↓
玩家掷 D20
    ↓
├── 点数 ≥ 难度 → 事件按意图发生
└── 点数 < 难度 → 事件以"搞耍"形式发生
```

**颠覆性评估：** 调用LLM对玩家输入的事件描述进行评分（1-10），依据以下维度。评估prompt固定，不随游戏状态变化。

| 维度 | 加分 |
|------|------|
| 涉及死亡/严重伤害 | +3 |
| 改变NPC核心动机 | +2 |
| 影响超过3个NPC | +1 |
| 涉及稀有物品/隐藏信息 | +1 |

**事件注入实现：** 骰子判定后，调用LLM将事件（原版或搞耍版）描述为可在世界状态中执行的具体效果，更新 world.md，并通过Orchestrator广播给受影响NPC。

**骰子对照表（D20）：**

| 颠覆性 | 难度 | 所需点数 |
|--------|------|---------|
| 1-2 | 极低 | ≥ 3 |
| 3-4 | 低 | ≥ 8 |
| 5-6 | 中 | ≥ 12 |
| 7-8 | 高 | ≥ 16 |
| 9-10 | 极高 | ≥ 19 |

**搞耍式失败：**
事件仍然发生，但以荒诞、打折或扭曲的形式。失败不代表浪费——搞耍结果本身成为剧情的一部分，写入世界记忆，NPC可能在后续对话中提起。

例：
- 玩家要求"农夫持刀袭击警探"→ 颠覆性9 → 需要≥19 → 骰子失败
- → 搞耍："农夫持刀冲向警探，但刀是伸缩玩具刀。警探愣了一下：'干嘛呀你！'全场哄笑。"

## 3. NPC自主交互系统

### 3.1 三层触发

```
NPC自主交互触发：
│
├── 时段驱动
│   18:00-22:00 酒馆社交 —— 所有NPC聚集，随机配对3-5轮对话
│   其他时段 —— 按日程表偶遇和工作交互
│
├── 状态驱动
│   mood < 20 → 可能主动找信任的NPC倾诉
│   hunger < 20 → 可能向他人讨要食物
│   存在未解决矛盾 → 可能爆发冲突
│
└── 事件驱动
│   玩家注入事件 → NPC反应 → 连锁讨论
│   过去事件残留 → 可能在数天后突然被提及
```

### 3.2 意外剧情的诞生路径

```
路径1：NPC A喝多了 → 对B说漏嘴一个秘密
路径2：事件+状态共振 → 农夫mood低 + 酒保涨价 → 农夫在酒馆发火
路径3：连锁反应 → A对B的态度变化 → B找C诉苦 → C对A产生敌意
路径4：搞耍事件残留 → 两天后NPC突然提起"你上次那个玩具刀..."
```

玩家第二天醒来，可能发现村庄气氛已经变了——昨晚发生了什么，只能通过对话碎片去拼凑。

### 3.3 交互约束

- LLM生成对话内容，角色性格锚定（system prompt固定）
- 当前状态注入（数值映射为自然语言）
- 双方关系权重影响语气
- 关键交互结果写入 agent_mem.md 成为长期记忆
- NPC token耗尽时跳过该NPC的社交轮次，记录"沉默不语"

## 4. 秘密系统

### 4.1 秘密属性

```
秘密 {
    描述: str                    # 简短摘要
    持有者: NPC ID
    颠覆性: 1-10                 # 对玩家认知的冲击力
    来源: "预写" | "自然生长"     # 手工编写 or NPC交互产生
    关联秘密: [秘密ID]            # 网状连接
    知情者: [{NPC ID, 知情程度}]  # 哪些NPC知道（或部分知道）
}
```

### 4.2 混合来源

- **预写硬核秘密**：开发者为每个NPC手工编写1-2层核心秘密（质量保证）
- **自然生长秘密**：NPC自主交互产生的矛盾、意外暴露、新关系（鲜活感）

### 4.3 秘密解锁

- 好感度（即信任）越高，NPC在对话中越可能说漏嘴
- 颠覆性越高，所需好感度阈值越高
- NPC自主交互可能意外暴露秘密（完全自发，不需要玩家干预）
- 偷听是和发现秘密的主要协助手段

### 4.4 秘密交换

玩家可以向NPC透露其他NPC的秘密，换取新信息或更深层的秘密。透露有风险：
- NPC可能因此更信任你（感谢告知）
- NPC可能因此降低好感度（觉得你大嘴巴）
- 后果取决于：X和Y的关系 + 秘密的性质 + NPC的个性

## 5. 玩家能力与日常节奏

### 5.1 玩家能力

| 能力 | 规则 |
|------|------|
| 对话 | 始终可行，每游戏小时最多3次交互，同NPC间隔10游戏分钟 |
| 偷听 | 每天1次，不限时机（可跟踪NPC独处、可旁听私聊、可在酒馆选正在交谈的组合） |
| 发布事件 | 每天1次，随机抽取或自定义+骰子 |
| 秘密交换 | 任何时候可在对话中透露秘密 |

### 5.2 日常时间表

玩家需要根据NPC日程决定"什么时间、去什么地方、找谁"——偷听和私聊需要策略性安排。

| 时间段 | NPC状态 |
|--------|--------|
| 06:00-08:00 | 起床、准备 |
| 08:00-12:00 | 工作/日常活动 |
| 12:00-13:00 | 午餐休息 |
| 13:00-18:00 | 工作/日常活动 |
| 18:00-22:00 | 酒馆社交（所有NPC聚集，交互高峰） |
| 22:00-06:00 | 睡眠（大部分NPC不可交互） |

### 5.3 好感度系统

- 好感度即信任（单轨，简化设计）
- 好感度决定对话质量 + 秘密透露概率
- 好感度因"大嘴巴"行为而双向波动（某些NPC喜欢，某些反感）

## 6. 玩家笔记簿

一本**活的事件编年史**，自动记录摘要。只记录玩家目击或听说的内容（信息不对称），不记录NPC私下发生的事。

### 6.1 结构

```
📜 事件时间线
  Day 1  玩家注入了"流浪商人"随机事件
  Day 1  酒保和流浪商人做了交易（你在酒馆听到的）
  Day 2  玩家自定义事件"让乞丐捡到一枚金币"→ 骰子失败→ 乞丐捡到假金币
  Day 3  农夫在酒馆和酒保吵架（NPC自主——你不知道为什么）
  ...

👤 NPC观察日志（按NPC分组）
  酒保：
    - Day 1：和流浪商人交易了东西
    - Day 3：和农夫吵架，摔了一个杯子
  农夫：
    - Day 2：捡到了乞丐丢掉的假金币
    - Day 3：在酒馆发火，原因不明
  ...

💬 玩家行动记录
  Day 1 偷听了酒保和流浪商人的对话
  Day 2 告诉警长"酒保可能在做不干净的生意"
  Day 3 和农夫一起耕作，交谈中得知他讨厌酒保涨价
```

### 6.2 设计规则

- 只记已发现的信息，留白不填充
- 记录是摘要化的，不保留完整对话
- 玩家可手动添加"推测"标记，与客观事实区分

## 7. 无终点设计

- 没有固定结局条件
- NPC不断产生新的秘密和关系变化
- 玩家玩到不想玩为止，存档保留随时可回来
- 玩家可随时对NPC说"我要走了"，NPC根据关系深度给出告别反应

## 8. 第一天体验

- 完全陌生，没有引导，没有教程
- 玩家走进村庄，不知道任何人名、任何事
- 从第一句对话开始自己摸索世界

## 9. 与现有 v1 设计的差异

| 维度 | v1（当前规格） | v2（本设计） |
|------|--------------|------------|
| 核心驱动力 | 通过对话与NPC交互推动剧情 | 观察NPC自主演化 + 注入事件搅局 |
| 玩家角色 | 村庄居民，与NPC交友 | 隐形推手 + 观察者 |
| 剧情来源 | 预设结局条件（大暴动、警徽、农夫乐事等） | NPC自主交互 + 玩家事件注入产生意外剧情 |
| 结局 | 多种固定结局 | 无终点，持续探索 |
| 秘密系统 | 无 | 洋葱式秘密网（预写+自然生长） |
| 事件系统 | 有限随机事件（仅世界级） | 每日事件特权（随机+自定义+骰子） |
| 偷听 | 无 | 每天1次，自由时机 |
| 笔记簿 | 无 | 自动摘要事件时间线+NPC观察日志 |
| NPC自主交互 | Phase 2 才引入 | 核心引擎，Day 1就存在 |
| 信任体系 | 好感度 | 好感度即信任（单轨） |

**保留的 v1 设计：**
- 技术架构（Vue 3 + FastAPI + DeepSeek + JSON存储）
- NPC状态系统（health/hunger/fatigue/mood）
- 时间表结构、交互配额、Token预算控制
- 预运行、存档机制、多用户数据结构
- NPC身份配置（背景设定文本结构化字段）
- 记忆管理（user.md / agent_mem.md / self.md）
